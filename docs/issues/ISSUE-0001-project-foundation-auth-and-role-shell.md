## What to build

Behavior delivered:
- Student registration/login with allowed institutional email domain policy.

- Role guards distinguish Student, Staff, and Super Admin shells.
- Inactive users cannot log in or refresh sessions.
- Student, Staff, and Admin shells are reachable by the correct role.

Architecture changes:
- Added UserAccountModule to concentrate registration, login, refresh, token resolution, and account error modes.
- Added AccessPolicyModule for role-limited access checks behind a dedicated Interface.
- Added SettingsModule for runtime configuration defaults, environment loading, and allowed-domain normalization.
- Added UserRepository Interface with SqlAlchemyUserRepository Adapter as the User persistence Seam.
- Added HttpApplicationModule for FastAPI route registration and dependency assembly; create_app is now a small entry point Adapter.

Documentation and tests:
- Added CONTEXT.md domain vocabulary for User Account, Access Policy, Application Settings, User Repository, and HTTP Application.
- Added behavior tests for auth foundation, access policy, settings, user repository, and HTTP application assembly.
