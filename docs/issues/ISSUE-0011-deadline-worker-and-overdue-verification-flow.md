## Agent Brief

**Category:** enhancement
**Summary:** Add time-based reservation deadline processing and completed-status handling.

**Current behavior:**
The reservation lifecycle now includes reservation submission, generated approval letters, signed-letter review, and paid receipt review. However, reservation deadline fields and worker processing are not yet fully represented in issue tracker state. Without the deadline worker behavior, student-caused missed uploads, staff-caused verification delays, final approval cutoffs, and completed status persistence are not centrally processed.

**Desired behavior:**
The system should track reservation deadlines across the document and payment workflows, then process them through a backend worker/service. Student-caused missed upload deadlines should expire the reservation. Staff-caused missed verification deadlines should move the reservation into an overdue verification state, preserving the slot until the shorter staff-overdue cutoff. Approved reservations whose end time has passed should be treated as completed even before worker persistence, and the worker should persist completed status after end time.

**Key interfaces:**
- Reservation status model — needs an overdue verification state and completed handling.
- Reservation deadline fields — should represent document upload, document verification, payment upload, and payment verification due times.
- Reservation workflow services — should set deadline fields when reservations are created, signed letters are uploaded, paid document approval opens payment, and payment receipts are uploaded.
- Deadline worker/service — should process due reservations based on current time and booking settings.
- Student reservation read behavior — should project approved past reservations as effectively completed even before the worker persists completion.

**Acceptance criteria:**
- [ ] Reservations expire when student document upload deadlines are missed.
- [ ] Reservations expire when student payment upload deadlines are missed.
- [ ] Staff-caused missed document verification deadlines move reservations into overdue verification.
- [ ] Staff-caused missed payment verification deadlines move reservations into overdue verification.
- [ ] Normal final approval cutoff is enforced at 7 days before reservation start by default.
- [ ] Staff-overdue final approval cutoff is enforced at 4 days before reservation start by default.
- [ ] Approved past reservations are returned as effectively completed even before worker persistence.
- [ ] Worker processing persists completed status after approved reservation end time.
- [ ] Implementation follows TDD with one behavior test at a time.

**Out of scope:**
- In-app notifications for deadline transitions.
- Scheduler deployment/runtime wiring.
- Email, WhatsApp, SMS, or push notifications.
- Revising document/payment review rules outside deadline state transitions.
