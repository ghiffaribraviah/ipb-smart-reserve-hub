## Agent Brief

**Category:** enhancement
**Summary:** Add Super Admin review moderation and immutable audit log visibility.

**Current behavior:**
Students can submit one review for their own completed reservation and can soft-delete their own reviews. Public facility pages and staff read-only review/statistics views exclude soft-deleted reviews. Super Admin users do not yet have a global review moderation view, cannot remove or restore reviews, and there is no audit log model/API for important workflow actions.

**Desired behavior:**
Super Admin users should be able to view all facility reviews globally with useful filters, including active reviews, student-deleted reviews, and admin-deleted reviews. Super Admin users should be able to soft-delete a review with a required reason and restore a soft-deleted review. Super Admin users must not be able to edit review content or rating.

When a Super Admin removes a review, the required removal reason should be stored on the review record so the student can see that their review was removed by admin and why. The audit log should also record the admin removal action, but it does not need to duplicate the removal reason.

Important workflow actions should create immutable, append-only audit log entries. Super Admin users should be able to view audit logs with filters for actor, action type, target type, facility, student, reservation, and date range.

**Key interfaces:**
- Review moderation workflow/API — lists all reviews globally for Super Admin, supports filtering, supports admin soft deletion with a required reason, supports restoration, and exposes no edit behavior for review content or rating.
- Student review read contract — lets a student see when their review was removed by Super Admin and the stored removal reason.
- Review persistence contract — distinguishes student soft deletion from Super Admin moderation removal strongly enough that global moderation, public visibility, student visibility, and restoration behave correctly.
- Audit log workflow/API — records immutable append-only entries and exposes Super Admin read access with the required filters.
- Workflow action emitters — create audit entries for review created, student soft-deleted review, Super Admin soft-deleted review, Super Admin restored review, document approval/rejection, payment approval/rejection, cancellation approval/rejection, staff assignment changes, and deadline expiry actions.
- Access policy — Super Admin-only moderation and audit-log access should use named access-policy actions rather than raw role checks.

**Acceptance criteria:**
- [ ] Super Admin can list all reviews globally, including active, student-deleted, and admin-deleted reviews.
- [ ] Super Admin review listing supports useful MVP filters for review moderation.
- [ ] Super Admin can soft-delete a review only when a non-empty reason is provided.
- [ ] Super Admin soft deletion stores the removal reason on the review record.
- [ ] Students can see when their own review was removed by Super Admin and why.
- [ ] Super Admin can restore soft-deleted reviews.
- [ ] Super Admin cannot edit review rating or comment through any moderation behavior.
- [ ] Public facility review lists, rating averages, and review counts continue to exclude deleted or admin-removed reviews.
- [ ] Staff read-only facility review/statistics views continue to exclude deleted or admin-removed reviews and expose no moderation commands.
- [ ] Audit logs are immutable and append-only, with no update or delete behavior exposed.
- [ ] Audit entries are created for review created, student soft-deleted review, Super Admin soft-deleted review, Super Admin restored review, document approval/rejection, payment approval/rejection, cancellation approval/rejection, staff assignment changes, and deadline expiry actions.
- [ ] Super Admin can view audit logs filtered by actor, action type, target type, facility, student, reservation, and date range.
- [ ] Non-Super Admin users cannot access Super Admin review moderation or audit-log views.
- [ ] Implementation follows TDD with one behavior test at a time.

**Out of scope:**
- Editing review content or rating.
- Staff review moderation.
- Review approval workflows.
- Notifications, reminders, or automated moderation prompts.
- Duplicating the Super Admin removal reason inside the audit log payload when it is already stored on the review record.
- Changing reservation completion rules beyond what existing review eligibility and workflow audit events require.
